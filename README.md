# archive-portal-index

本仓库用于归档和发布多个独立 HTML 页面，不针对任何特定域名或具体网站。主要目的是提供一个集中存放、版本管理和预览的索引空间，方便维护和分发静态内容。

## 项目简介

该仓库存储了一系列独立 HTML 文件，每个文件代表一个独立的页面。这些页面可能用于不同的用途，但统一托管于此以便于管理和存档。仓库本身不包含动态逻辑或后端服务，仅作为静态文件集合。

## 目录说明

```
.
├── index.html          # 仓库主索引页面（如有）
├── pages/              # 存放主要 HTML 页面文件的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面，文件名自解释。
- `assets/`：存放页面所需的公共资源，按类型分目录组织。
- 根目录下的 `index.html` 可作为入口或导航页（可选）。

## 页面归档说明

- 每个 HTML 页面应保持独立，不依赖仓库外的资源（除非明确说明）。
- 页面内可引用 `assets/` 目录下的资源，建议使用相对路径。
- 归档时请确保页面功能完整，无外部链接依赖。
- 页面文件名应清晰描述内容，避免使用无意义命名。

## 维护说明

- 本仓库采用常规 Git 管理，提交信息应简要说明变更内容。
- 添加新页面时，请将文件放入 `pages/` 目录，并更新相关索引（如有）。
- 删除或修改页面时，请确认无其他页面依赖该文件。
- 不建议在页面中嵌入敏感信息或动态服务，保持静态特性。

## 使用方式

1. 克隆仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件即可预览。
3. 也可通过 GitHub Pages 或其他静态托管服务部署整个仓库。

## 贡献

欢迎提交 Pull Request 添加或更新页面。请确保：
- 页面内容合法合规。
- 不包含特定域名或诱导性内容。
- 遵循上述目录结构和归档规范。
